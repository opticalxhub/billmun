ROLES IN THIS SYSTEM
There are eight roles: DELEGATE, CHAIR, ADMIN, MEDIA, SECURITY, EXECUTIVE_BOARD, SECRETARY_GENERAL, DEPUTY_SECRETARY_GENERAL. Every role has its own dashboard experience. SECRETARY_GENERAL and DEPUTY_SECRETARY_GENERAL have full EB access plus special designation. ADMIN is a committee assistant — assigned to one specific committee only, assists the chair, sees only their committee's data. SECURITY is a new department responsible for physical conference security — badge checking, room access, incident management, delegate tracking for safety purposes. Every role is enforced at the middleware level on every API route and every page route.
 
DELEGATE DASHBOARD — /dashboard/delegate
The delegate dashboard is the delegate's complete conference workspace. Every tab, every section, every button is fully functional.
The dashboard has a top navigation tab bar with these tabs: Overview, My Committee, Documents, AI Feedback, Speeches, Blocs, Resolution Builder, Schedule, Research. On mobile this tab bar is horizontally scrollable. The active tab is visually distinct. Each tab loads its content below without a full page reload.
Overview Tab
A full-width live committee status banner at the top. Shows committee name, assigned country with flag emoji, committee topic, chair name, and a live session status badge. Status options are In Session, Moderated Caucus, Unmoderated Caucus, On Break, Adjourned. When In Session a pulsing dot appears. When Moderated Caucus the current debate topic and speaking time limit appear. This refreshes every 20 seconds via polling and updates instantly via WebSocket when the chair changes it. Three buttons below the banner: Download Background Guide, Download Rules of Procedure, View Roster. Roster opens a modal table showing every committee delegate with name, country, online status, and paper submitted indicator.
Below the banner, a row of four stat cards: Documents Submitted showing the count of all documents the delegate has uploaded, AI Analyses Used Today showing X of 10, Speeches Drafted showing total draft count, Blocs Joined showing total bloc membership count. On tablet these are 2x2. On mobile they stack to single column.
Below the stats, a two-column layout on desktop — left column showing Recent Activity as a chronological timeline of everything the delegate has done on the portal with timestamps, right column showing a Conference Countdown counting down days hours minutes seconds to the conference date from ConferenceSettings, and below it a Quick Actions card with large tap-friendly buttons for Upload Document, Start AI Analysis, View Committee, Open Messages. On tablet the columns stack. On mobile single column.
My Committee Tab
Everything about the delegate's committee in one place. The committee full description. The committee topic with a detailed breakdown. The background guide displayed as an embedded PDF viewer with a download button. The rules of procedure the same way. A committee member roster table showing all delegates with name and country — on mobile this is a card list. The current session status banner identical to the overview. Any announcements posted specifically to this committee appear as cards at the top of this tab. The chair's contact — a Direct Message button opens a DM to the chair.
Documents Tab
A complete document management workspace. A drag-and-drop upload zone accepting PDFs up to 10MB. On mobile this is a large tap target with a Choose File button as fallback. Real Uploadthing upload with a visible progress bar. On completion the document appears immediately in the table below. The table shows title, type badge, committee, upload date, file size, status badge, reviewer name, review date. On mobile this table becomes a stack of cards each showing all the same information. Clicking any document opens a drawer — on desktop a right-side panel, on mobile a full-screen slide-up sheet. Inside: native PDF preview via iframe, full metadata, complete status change history as a vertical timeline, feedback from chair in a styled blockquote. If status is REVISION REQUESTED a banner shows the revision note and an Upload Revised Version button. All versions accessible in a Versions tab. Rename and delete controls where permitted. Download button. Every status change sends in-portal notification and email.
AI Feedback Tab
A text area where the delegate pastes any text or selects from their uploaded documents via a dropdown. An Analyze button sends to the Anthropic API. While processing, five messages cycle with fade transitions: Reading your argument structure, Evaluating policy alignment, Cross-referencing research depth, Drafting inline annotations, Finalizing your feedback report. A progress bar fills slowly beneath the messages. Results render in a full panel. Large circular overall score ring showing the percentage. Six smaller sub-score rings for Argument Strength, Diplomatic Language, Structural Clarity, Policy Alignment, MUN Format Adherence, Persuasiveness. Three columns of insight cards — Strengths, Weaknesses, Suggestions — each containing a specific observation quoting the relevant passage. An annotated document view renders the full text with highlighted spans at varying opacities — hover reveals a tooltip with the AI comment. An AI Detection section estimates likelihood the document was AI-generated. All results stored in database. Past Analyses section lists previous sessions expandable to full results. Rate limited to 10 per day server-side. Usage counter shows X of 10 used today. On mobile the score rings stack vertically. The three insight columns stack to single column.
Speeches Tab
A speech writing environment. Left panel on desktop is a scrollable list of all drafts — title, word count, speaking time estimate, last saved. On mobile this is a dropdown selector at the top. Right panel or full screen on mobile is the editor. Title field, large textarea for speech body. Live word count below the textarea. Live estimated speaking time at 130 words per minute shown as minutes and seconds. Autosave every 30 seconds with a brief Saved indicator. Manual Save button. Notes field below the textarea private to this delegate. Mark as Active Speech designates one draft as primary. Duplicate, Rename, Delete on each draft. Print opens printer-friendly layout. Copy to Clipboard copies full text.
Blocs Tab
All blocs the delegate belongs to as cards showing name, member count, last activity, last message preview. Create Bloc button opens a modal asking for name and description — generates a 6-character invite code shown on creation. Join Bloc button opens modal with invite code input. Clicking a bloc navigates to the full bloc workspace page. The bloc workspace has four tabs: Members, Documents, Strategy Board, Messages. Members tab: table of all members with name, country, online dot, join date — on mobile a card list. Creator sees Remove Member and Transfer Leadership. Invite code shown with Regenerate button. Documents tab: shared file library — upload, see all members' uploads, download any file. Strategy Board tab: large shared textarea syncing via polling every 10 seconds — all members see each other's changes. Below it each member has a private individual section only they write in. Messages tab: the full real-time messaging channel for this bloc — WhatsApp style, file sharing, reactions, replies, typing indicators, all working.
Resolution Builder Tab
Create and manage multiple resolution drafts. Each has title, topic, committee, and co-sponsors as tags. Editor has two sections: Preambulatory Clauses and Operative Clauses. Adding a clause shows a dropdown of all standard MUN opening phrases for that clause type — full list of preambulatory and operative starters. After selecting the phrase the rest of the clause is typed inline. Drag-and-drop reorder within each section. Sub-clauses under operative clauses at one indentation level. On desktop a live preview panel on the right renders the resolution formatted as an official MUN document — header, forum, question of, sponsors, signatories, preambulatory clauses italic, operative clauses numbered with underlined opening phrase. On mobile the preview is a toggle. Download as PDF generates the formatted document. Export as Word generates DOCX. Share to Bloc sends the document as a file to any of the delegate's bloc channels.
Schedule Tab
Official conference schedule from EB displayed as a day-by-day timeline. Each event shows name, location, time range, description, applicable roles. Below the official schedule, personal tasks — add a task with title, optional notes, due date and time, completion checkbox. Overdue unchecked tasks are visually distinct. Conference countdown widget at the top. On mobile the timeline is a vertically stacked card list.
Research Tab
Private research workspace. A Country Profile section with a textarea for research notes autosaved every 30 seconds. A Stance Tracker — for each committee sub-topic a label and textarea for the country's position, saved independently. A Previous Resolutions section to paste or note relevant past UN resolutions for reference.

Speakers List Tab

Active speaker highlighted prominently at top with Now Speaking and speaking timer running. Next Speaker advances queue and starts speaking timer. Yield Time marks speaker as yielding remaining time to another delegate or to the chair. Yield to Questions opens a 30-second window for points of information. The chair logs each point or motion as it is raised. Type selector: Point of Order, Point of Information, Point of Personal Privilege, and all standard MUN motions. Delegate selector from committee roster. Outcome: Passed, Failed, Ruled Out of Order, Withdrawn. Passed motions affecting session state auto-trigger the corresponding status change. All entries stored in database as part of the session record. A running session log compiles every event chronologically — status changes, motions, speakers, timer events. Documents Tab
Full document review system. Table of every document submitted to this committee — delegate name, title, type, submission date, status badge. On mobile a card list. The admin dashboard has a tab bar: Overview, Delegate Logistics, Attendance, Documents, Announcements, Resources, Session Support, Communication.

Header showing which committee they assist and the chair's name. A Live Session Status card showing the current committee status, delegate count present versus total, current debate topic. A Quick Message button to DM the chair directly. Pending tasks summary — documents awaiting review, delegate status alerts, unread committee messages.
Delegate Logistics Tab
The core admin feature. Table of every approved delegate in this committee — name, country, current physical status, last status change, who recorded it. Physical statuses: Present In Session, Present Unmoderated Caucus, Lavatory Break, Medical Break, Missing, Absent with Reason, Absent without Reason. Each row has an inline status dropdown that saves immediately on change. Saving logs the change to audit log with admin id and timestamp. Clicking a delegate name opens a status history panel showing every status change for the entire conference. A visual summary bar at the top shows real-time counts of each status. An Alert system — any delegate on Lavatory Break or marked Missing for more than 15 minutes gets a flag on their row and a notification sent to the EB channel and the chair. On mobile each delegate is a card with a large status selector.

Communication Tab

SECURITY DASHBOARD — /dashboard/security
The Security role is the conference's physical security department. Security personnel manage badge access, monitor delegate movement, handle incidents, and coordinate with EB on safety matters. The security dashboard has a tab bar: Overview, Badge Control, Incident Reports, Delegate Tracker, Access Zones, Communication.
Overview Tab
A live operational overview. Total delegates currently checked in as a large stat card. Total incidents reported today. Open incidents requiring attention. Active access zones and their current occupancy. A live activity feed showing every security-relevant event — delegate checked in, incident reported, access zone status changed, badge flagged — in real time via WebSocket. A quick action row: New Incident Report, Check In Delegate, Flag Badge, Broadcast Security Alert. On mobile these are large tap-friendly cards.
Badge Control Tab
The core security feature. A search field at the top — type any delegate name, email, or badge number to pull up their record instantly. The delegate record shows full name, photo if uploaded, role, committee, country, badge number, badge status (Active, Suspended, Lost, Flagged), and check-in history. Action buttons: Check In marks the delegate as present and logs the timestamp and location. Check Out marks them as departed. Suspend Badge — requires reason, immediately flags the badge in the system and notifies EB. Mark Lost — creates a lost badge record and triggers a replacement workflow. Flag for Review — marks the badge for secondary review without suspending. A full badge log table below the search shows the 50 most recent badge events across all delegates with name, action, location, timestamp, and security officer who performed the action. A Bulk Check-In mode shows all delegates in a scrollable list with a single Check In button per row — designed for quickly processing a group entering a room. On mobile the search and delegate card take full width with large action buttons.
Incident Reports Tab
Create and manage security incident reports. New Incident Report button opens a full form: incident type dropdown (Unauthorized Access, Disturbance, Medical, Missing Person, Suspicious Activity, Property Issue, Other), location field, involved parties field (select from delegate roster), description textarea, severity level (Low, Medium, High, Critical), immediate action taken field, does this require EB notification toggle. Submitting a Critical or High severity incident automatically sends a notification to all EB members and the Secretary General. The incidents table shows all reports — incident ID, type, severity badge, location, reported by, reported time, status (Open, In Progress, Resolved, Escalated). Clicking a report opens the full detail view with all fields, the ability to add follow-up notes, change status, and assign to another security officer. Resolved incidents require a resolution note. On mobile the table is a card stack with expandable detail.
Delegate Tracker Tab
A real-time view of delegate physical locations within the conference venue. A simplified venue map — a visual grid or floor plan layout where each room or zone is a labeled block. Each zone shows its name, capacity, and current occupancy count. Clicking a zone shows a list of all delegates currently logged as being in that zone. A security officer can manually move a delegate from one zone to another by selecting the delegate and selecting their new zone. Movement is logged to the audit log with timestamp and officer name. A delegates-at-risk section flags any delegate who has not been seen or checked in for more than 30 minutes during conference hours. A Missing Persons tracking sub-section — formally log a delegate as missing with last known location and time, this triggers an immediate notification to all security officers and EB. On mobile zones are a scrollable card list.
Access Zones Tab
Manage the conference venue's access zones. A list of all defined zones — zone name, description, authorized roles, current status (Open, Restricted, Closed). Each zone has an authorized roles multi-select — which roles are allowed to enter. Changing a zone's status to Restricted logs the reason and notifies EB. Changing to Closed requires EB approval. Security officers can add new zones, edit existing ones, and view a complete access log for any zone showing every delegate who entered and exited with timestamps. A zone occupancy history chart shows peak and low usage times per zone.
Communication Tab
Direct access to the security team's internal communication channel. A large prominent button to send a Security Alert — a broadcast message to all security officers simultaneously. The alert has a severity level and message. A direct message shortcut to the Secretary General. A direct message shortcut to any EB member. A Security Briefing section — security team leader can post briefings visible to all security personnel with read receipts showing who has seen them. On mobile all communication elements are full width with large tap targets.
