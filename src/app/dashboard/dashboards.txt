ADMIN ROLE CLARIFICATION — READ THIS FIRST
The Admin role is not a global administrator. Admins are Committee Assistants — each admin user is assigned to one specific committee by the Executive Board and they assist the chair of that committee only. They can see and act on data only within their assigned committee. They have no access to other committees, no global settings, no registration management. Think of them as the chair's right hand during the session — handling logistics, tracking delegates, managing documents, so the chair can focus on running debate. The EB dashboard is the only place global administration happens. This distinction must be enforced at the middleware and API level — every admin API route checks that the requested resource belongs to the admin's assigned committee before returning any data.

DELEGATE DASHBOARD
The delegate dashboard is the delegate's complete conference workspace. It is not a summary page — it is the primary environment they live in for the entire conference.
Live Committee Status Banner
The top of the dashboard is a full-width banner showing the delegate's committee name in large text, their assigned country with flag emoji, the committee topic in smaller text below, and the chair's full name. On the right side of the banner, a live session status indicator shows exactly what is happening in the committee room right now — In Session, Moderated Caucus, Unmoderated Caucus, On Break, or Adjourned. When the committee is In Session, a small animated pulse dot appears next to the status. When in Moderated Caucus, the current debate topic and speaking time limit appear below the status. This data auto-refreshes every 20 seconds via polling and also updates instantly via WebSocket when the chair changes the session status. Three buttons sit below the banner: Download Background Guide pulls the committee's background guide file from Uploadthing and triggers a browser download. Download Rules of Procedure does the same for the ROP file. View Committee Roster opens a modal showing every delegate in the committee as a table with their name, country, online status dot, and a paper submitted indicator showing whether they have submitted their position paper yet.
Position Paper and Document Management
A full document management workspace. A drag-and-drop upload zone that accepts PDF files up to 10MB. The zone shows a dashed border that changes to solid when a file is dragged over it. Dropping triggers a real upload via Uploadthing with a progress bar that fills in real time. On completion the document appears instantly in the document table below without any refresh. The table shows every document the delegate has ever submitted — title, type badge, committee, upload date, file size in human-readable format, status badge, last reviewer name, and review date. Clicking any row opens a right-side drawer. Inside the drawer: a native PDF preview via iframe taking the top 55% of the panel. Below the preview: document title, type, committee, upload date, file size, current status. A Status History section shows every status transition as a vertical timeline with the new status, who changed it, and the timestamp. A Feedback section shows any feedback written by the chair or admin in a styled blockquote with the reviewer's name and when it was written. If the status is REVISION REQUESTED, a prominent banner shows the revision note and an Upload Revised Version button that creates a new document version linked to the original — all versions are shown in a Versions tab with download links. The delegate can rename a document if it is in SUBMITTED status. They can delete it only if it is SUBMITTED and has no feedback written. A Download button downloads the file directly. Every status change anywhere sends an in-portal notification and a Resend email to the delegate.
Speech Preparation Workspace
A dedicated speech writing environment. The delegate creates unlimited speech drafts. Each draft has a title field at the top, a large textarea for the speech body, a live word count updating on every keystroke, and a calculated estimated speaking time at 130 words per minute shown as minutes and seconds. An autosave runs every 30 seconds showing a brief Saved indicator that fades after 2 seconds. A manual Save button is also present. All drafts appear in a scrollable panel on the left side — each entry shows the title, word count, speaking time, and last saved timestamp. Clicking a draft loads it into the editor. A Mark as Active Speech button highlights one draft as the current primary speech in the list. A Notes field below the main textarea is a private scratchpad for research points and reminders visible only to the delegate. Duplicate, Rename, and Delete actions on each draft. A Print button opens a clean printer-friendly layout. A Copy to Clipboard button copies the full text.
AI Analysis Engine
The delegate selects a document from a dropdown of their uploaded position papers or pastes text directly into a large textarea. An Analyze button sends the content to the Anthropic API. While processing, five cycling status messages fade in and out one at a time: Reading your argument structure, Evaluating policy alignment, Cross-referencing research depth, Drafting inline annotations, Finalizing your feedback report. Below the messages, a thin progress bar slowly fills over an estimated 12 seconds. Results render in a panel with a large circular overall score ring in the center, then six smaller rings for Argument Strength, Diplomatic Language Quality, Structural Clarity, Policy Alignment, MUN Format Adherence, and Persuasiveness. Below the rings, three columns of specific insight cards — Strengths, Weaknesses, Actionable Suggestions — each card a specific observation about the submitted text with a relevant quoted passage. Below the columns, an annotated document view renders the full submitted text with spans highlighted at different opacity levels reflecting quality — hovering any highlighted span shows a floating card with the AI's specific comment about that exact passage. A separate AI Detection Check estimates the likelihood the document was AI-generated and flags suspicious phrasing patterns. All results stored in AIFeedback table. A Past Analyses section lists all previous sessions expandable to full results without re-running the API. Rate limited to 10 analyses per day enforced server-side via database counter resetting at midnight UTC. A usage indicator shows X of 10 analyses used today.
Blocs and Diplomatic Groups
All blocs the delegate belongs to appear as cards showing bloc name, member count, last activity, and a last message preview. A Create Bloc button opens a modal asking for a bloc name and optional description — on creation a 6-character uppercase invite code is generated and displayed. A Join Bloc button opens a modal with an invite code input field. Clicking a bloc card navigates to the full bloc workspace which has four tabs. Members tab: table of all members with name, country, online status dot, and join date. The bloc creator sees a Remove Member button and a Transfer Leadership button. An Invite Code section shows the current code and a Regenerate Code button. Documents tab: shared file library where any member can upload files — each shows uploader name, upload date, size, and a Download button. Strategy Board tab: a large shared textarea where any member can type and all members see changes via polling every 10 seconds — this is the shared diplomatic scratchpad for collective position and resolution points. Below it, each member has a private individual contribution area only they can write in. Messages tab: the full real-time bloc channel from the messaging system — WhatsApp-style with file sharing, reactions, replies, and typing indicators.
Resolution Builder
The delegate can create and manage multiple resolution drafts. Each has a title, topic, committee, and a sponsors field accepting country name tags. The editor has two sections: Preambulatory Clauses and Operative Clauses. Adding a clause shows a dropdown of all standard MUN opening phrases for that clause type. After selecting the opening phrase the delegate types the remainder of the clause in the same input. Clauses reorder via drag-and-drop within their section. Sub-clauses can be added under any operative clause at one level of indentation. A live preview panel on the right renders the resolution formatted exactly as an official MUN document — header, forum, question of, sponsors, signatories, preambulatory clauses in italics, operative clauses numbered with underlined opening phrase. Download as PDF generates the formatted document. Export as Word generates a DOCX. Share to Bloc sends the document as a file to any of the delegate's bloc channels.
Country Research Dashboard
A private research workspace for the delegate's assigned country. A Country Profile section shows the country name and a text area where the delegate writes and saves their research notes — autosaved every 30 seconds. A Stance Tracker lets the delegate record their country's position on each sub-topic of the committee's main agenda item — for each sub-topic there is a label and a textarea for the position, all saved independently. A Previous Resolutions section where the delegate can paste or note relevant past UN resolutions for quick reference during debate.
Personal Conference Schedule
Shows the official conference schedule uploaded by EB as a day-by-day timeline. Each event shows its name, location, time range, description, and which roles it applies to. Below the official schedule, a personal tasks section where the delegate adds their own reminders with a title, optional notes, due date and time, and a completion checkbox. Overdue unchecked tasks are visually distinct. A conference countdown widget shows days, hours, minutes, and seconds until the conference start date live from ConferenceSettings.
Personal Statistics
Total documents submitted, AI analyses run today and all time, speeches drafted, blocs joined, messages sent this conference. A chronological activity timeline showing every action the delegate has taken on the portal with timestamps — uploaded document, joined bloc, ran analysis, sent message, updated speech. This is the delegate's permanent conference record.

CHAIR DASHBOARD
The chair dashboard is the command center for running a live committee session. Every single feature here is a real operational tool.
Committee Command Header
At the top, the committee name large, the committee topic below it, and a large Session Status badge showing the current state with a pulse indicator when In Session. A prominent Session Status control lets the chair change the status to In Session, Moderated Caucus, Unmoderated Caucus, On Break, or Adjourned. Changing to Moderated Caucus opens a modal asking for the debate topic and per-speaker time limit — both are saved and broadcast via WebSocket to all committee delegates immediately. Changing to Unmoderated Caucus asks for duration and purpose. Changing to Break asks for break type and estimated duration. Changing to Adjourned asks for a brief session summary note. All status changes are timestamped and logged in the database. A delegate count badge shows present versus total assigned pulled from the latest roll call.
Roll Call System
A full digital roll call interface. A table of all assigned delegates with name and country. Three states per delegate: Absent, Present, Present and Voting — the chair cycles through them by clicking. A Start Roll Call button locks the interface into roll call mode and starts a visible elapsed timer. A Complete Roll Call button finalizes the record saving it to the database with a session timestamp. A quorum indicator calculates whether quorum is met based on the configured threshold from ConferenceSettings and displays either Quorum Established or Quorum Not Reached with exact counts. All previous roll calls are accessible via a Roll Call History button showing every past session's record as a table with date, attendance percentage, and quorum status. Attendance data feeds the delegate performance tracker.
Session Timer
A large digital countdown clock displayed prominently. The chair inputs time in minutes and seconds. Start, Pause, Resume, and Reset buttons. When the timer expires it flashes visually and fires a browser notification if permissions are granted. An Add 30 Seconds button extends the running timer instantly. A label input lets the chair tag what the timer is currently timing. A complete log of every timer run during the current session is shown below — label, set duration, actual duration if stopped early, and start time. A dedicated Speaking Timer mode starts automatically when the chair marks a speaker as Now Speaking in the speakers list and counts down from the configured speaking time.
Speakers List
A full queue management system. The chair adds delegates from a dropdown of all committee delegates or types a name. The queue shows position number, delegate name, country, and estimated time until their turn based on current speaking time. Drag-and-drop reorder the entire queue. The active speaker is prominently highlighted at the top with a Now Speaking label and their speaking timer ticking. A Next Speaker button advances the queue and starts the speaking timer. A Yield Time option marks the speaker as yielding remaining time to another delegate or to the chair. A Yield to Questions option opens a 30-second window for points of information which can be extended. The list tracks how many times each delegate has spoken this session. Per-delegate speaking statistics show total speaking time and speech count for every delegate in the committee. The chair can clear the entire list, export the session's speaking record as a PDF, and save and reload named speaker list templates.
Points and Motions Log
The chair logs points and motions as they are raised during debate. Adding an entry shows a type selector — Point of Order, Point of Information, Point of Personal Privilege, Motion to Open Debate, Motion to Close Debate, Motion to Table Agenda, Motion to Introduce a Working Paper, Motion to Move into Voting Procedure, Motion to Suspend the Meeting, Motion to Adjourn the Meeting, Motion for an Unmoderated Caucus, Motion for a Moderated Caucus. The chair selects which delegate raised it from the committee roster dropdown. An outcome is recorded — Passed, Failed, Ruled Out of Order, Withdrawn. Passed motions that affect session state automatically trigger the appropriate session status change. All entries are stored in the database as part of the official session record. A running session log tab compiles every event in chronological order — status changes, motions, speakers, timer events — and can be exported as a formatted PDF of official committee minutes.
Working Paper and Resolution Review
A complete document review system for all documents submitted to this committee. A table shows every document across all committee delegates — delegate name, document title, type badge, submission date, status badge. Clicking a document opens a full review panel with a PDF preview via iframe, a status dropdown to change to Under Review, Approved, Revision Requested, or Rejected, a feedback textarea, and a Submit Review button that saves and immediately notifies the delegate. The chair can see all versions of a document in a Versions tab. When a working paper is approved it moves to an Approved Working Papers section which is accessible to all delegates in the committee as a shared resource. The chair can select multiple approved working papers and click Consolidate to create a new RESOLUTION_DRAFT document record linking the source papers.
Delegate Performance Tracker
For each delegate the chair can record: speech count (auto-populated from the speakers list), total speaking time (auto-populated), argumentation quality rated 1 to 5, diplomacy rating 1 to 5, preparation rating 1 to 5, and private chair notes. These ratings are stored in the database and visible only to the chair and EB — never to the delegate. An aggregate table shows all delegates ranked by a participation score derived from speech count and chair ratings. A Best Delegate Nominees section where the chair selects up to three delegates with written justifications — these nominations are submitted to the EB and viewable in the EB dashboard. A Printable Performance Report generates a formatted PDF of the full committee's participation data.
AI Tools for Chairs
Three dedicated AI tools. Working Paper Analyzer — the chair pastes or selects any working paper or resolution draft and the AI returns likelihood of AI generation, diplomatic language score, clause structure analysis, preambulatory to operative ratio, suggestions for strengthening specific clauses, logical contradiction flags, and a list of missing elements. Debate Quality Analyzer — the chair pastes a debate transcript or summary and the AI returns an analysis of overall debate quality, identification of substantive versus procedural contributors, suggested topics to deepen discussion, and recommended next motions. Speech Evaluator — paste any delegate speech and get argument strength, diplomatic language quality, structure, persuasiveness, and procedural appropriateness flags. All results stored and accessible in a history log.
Bloc Monitoring
The chair sees a read-only list of all blocs formed by delegates in their committee — bloc name, creation date, member count, and the list of member names and countries. The chair cannot read any messages or documents inside blocs — only the existence and membership is visible. This is so the chair understands the diplomatic alliances forming before and during session.
Pre-Conference Preparation Board
Before the conference starts, the chair has a preparation workspace. A Research Board where the chair saves research notes about the committee topic organized by sub-topic in labeled sections. A Country Position Overview — a table of all assigned delegate countries where the chair can add notes about each country's expected stance. A Preparation Checklist with items: Background Guide uploaded, Rules of Procedure uploaded, Country assignments confirmed, Pre-reading sent to delegates, Opening speech drafted, First debate topic selected — each item toggleable, with overall completion percentage shown at the top.
Committee Analytics
Charts showing position paper submission rate over time, attendance percentage by session, speaking time distribution across delegates as a horizontal bar chart showing whether participation is evenly distributed or dominated by a few delegates, motion type frequency as a breakdown of what types of procedural motions have been raised, and working paper approval rate. A Participation Equality Score shows whether the bottom half of delegates by speaking time are getting a fair share.
Session Export
A Generate Session Report button compiles a complete committee session PDF at any point — committee name and session number, date and times, attendance from roll call, quorum status, agenda items, all motions raised with outcomes, speakers list with speaking times per delegate, all working papers introduced, resolutions moved, chair summary notes, and per-delegate participation statistics. Formatted as a professional MUN committee minutes document.
Embedded Committee Channel
A compact real-time messaging panel embedded in the chair dashboard showing the committee's WhatsApp-style channel without leaving the page. The chair sends regular messages and creates Announcements from an Announce button — announcements have a title and body, are styled differently in the channel, pinned at the top, and trigger push notifications to all committee delegates. An announcement history shows all announcements sent in this committee.

Session Support Tools
A set of quick tools to support the chair during live session. A Delegate Lookup — type any name or country and instantly see that delegate's current status, document submission status, speaking count this session pulled from the speakers list, and contact information. A Committee Seating Chart generator that creates a visual grid of delegate positions — the admin inputs seating assignments and a printable PDF layout is generated showing each seat with the delegate name and country placard. A Working Paper Tracker showing all working papers introduced in this committee — which delegate submitted it, who co-sponsors it, current status, and whether it has been formally introduced on the floor. A Voting Record section where the admin records the outcome of any formal votes taken during session — motion type, outcome, vote count for and against and abstentions, which delegates voted which way if a recorded vote was called.
Communication with Chair
A direct messaging shortcut to the committee's chair prominently placed in the dashboard so the admin can communicate quickly without leaving their workspace. A private admin-to-chair notes section — a shared notepad visible only to the admin and their assigned chair where operational notes can be left during session without cluttering the main channel.

MESSAGING SYSTEM — REQUIRED ON ALL DASHBOARDS
The messaging icon is in the fixed navigation bar for all roles. The messages page at /messages has a two-panel layout — left panel 320px showing all channels and DMs the user has access to, right panel showing the active conversation. The following channels are created automatically. One channel per committee — every delegate in that committee is a member, the chair is a member, assigned admin assistants are members, and all EB members are members. A Chairs channel with all chair-role users and EB members. An Admin Team channel with all admin users and EB members. A Media Team channel with all media users and EB members. An Executive Board private channel with only EB members. An All Conference channel where every approved user is a member but only EB members and Secretary General can send messages — all others are read-only. Bloc channels created per bloc as described in the delegate section.
Every channel supports real-time messaging via Pusher or Ably WebSocket. Messages appear instantly. Typing indicators show live. Online presence dots appear on all avatars. File sharing via Uploadthing supports images and PDFs up to 25MB. Message reactions with emoji. Reply threading with quoted original. Read receipts per message. Message search across all channels. Pinned messages per channel. Announcements as a special message type pinned at the top and triggering push notifications. Unread counts per channel summed to a badge on the nav bar icon. Toast notification on incoming message in an unviewed channel.
Access and permissions enforced server-side: delegates can only message in their committee channel and their blocs plus read All Conference. Chairs can message in their committee channel, the Chairs channel, and read All Conference. Admins can message in their assigned committee channel, the Admin Team channel, and read All Conference. Media can message in the Media Team channel and read All Conference. EB can message in every channel and can post to All Conference.
